直近の変更をコードレビューしてください。

最初に `npm run check` を実行し、型エラーがあれば修正してからレビューを進めてください。

以下の観点でレビューし、問題があれば修正してください:

1. **型安全性**: `any` 型の使用、型アサーションが適切か
2. **入力バリデーション**: API境界での全フィールド検証
3. **セキュリティ**: CORS設定、SQLインジェクション、XSS
4. **エラーハンドリング**: fetchのタイムアウト設定、catch節の適切な処理
5. **Astro規約**: 静的/動的コンポーネントの使い分け
6. **コンテンツスキーマ**: Zod スキーマとの整合性
7. **パフォーマンス**: 不要な再レンダリング、重複fetch

レビュー対象:
- `git diff main` で変更差分を確認する
- 変更されたファイルのみをレビューする

出力形式:
- 問題点を重大度 (HIGH / MEDIUM / LOW) でラベリングして列挙する
- 修正すべき点は具体的なコードを示す
- 問題がない場合はその旨を明示する
